# TechCorp Information Security Policy

## Overview
This document outlines mandatory security practices for all TechCorp employees. Security is everyone's responsibility.

## Password Requirements

### Account Passwords
- Minimum 12 characters with complexity (uppercase, lowercase, numbers, special characters)
- Must use password manager (1Password provided by company)
- Enable multi-factor authentication (MFA) on ALL accounts
- Never share passwords via email, Slack, or text
- Rotate passwords every 90 days for critical systems

### MFA Requirements
All company accounts MUST have MFA enabled:
- Email (Google Workspace)
- Slack
- GitHub/GitLab
- AWS Console
- Any system containing customer data

Approved MFA methods: Authenticator apps (preferred), hardware keys, or SMS (least preferred).

## Device Security

### Laptops
- Full disk encryption required (FileVault for Mac, BitLocker for Windows)
- Auto-lock after 5 minutes of inactivity
- Keep OS and software up to date (install updates within 48 hours)
- Never leave laptop unlocked in public spaces
- Report lost/stolen devices to IT immediately (it-security@techcorp.com)

### Mobile Devices
- Use company MDM (Mobile Device Management) if accessing company email
- Screen lock with PIN/biometric required
- Do not store sensitive company data on personal devices

## Data Handling

### Classification Levels
1. **Public**: Marketing materials, public documentation
2. **Internal**: General company information, non-sensitive projects
3. **Confidential**: Financial data, customer PII, strategic plans
4. **Restricted**: Legal documents, security credentials, unreleased products

### Customer Data
- Never store customer data locally; use approved cloud storage only
- Access on need-to-know basis
- NEVER share customer data externally without legal approval
- Immediately report any suspected data breach to security@techcorp.com

### Data Sharing
- Use company-approved tools: Google Drive, Confluence, GitHub
- Avoid personal Dropbox, WeTransfer, or consumer file sharing
- Encrypt sensitive files before emailing (use company PGP keys)

## Network Security

### WiFi
- Always use company VPN when on public WiFi
- Avoid conducting sensitive work on public networks
- Home WiFi should use WPA3 or WPA2 encryption

### Phishing Prevention
- Verify sender before clicking links or downloading attachments
- Hover over links to preview URLs before clicking
- Look for signs: urgent language, spelling errors, unexpected attachments
- When in doubt, contact IT Security team
- Report suspicious emails using the "Report Phishing" button

## Acceptable Use

### Prohibited Activities
- Installing unauthorized software without IT approval
- Accessing competitors' systems or data unethically
- Sharing company credentials with contractors (create separate accounts)
- Using company resources for personal commercial projects
- Downloading pirated software or media

### Social Media
- Do not share unreleased product information
- Clearly state personal opinions are your own, not company's
- Report if your account is compromised

## Incident Response
If you suspect a security incident:
1. Disconnect device from network (if compromised)
2. Email security@techcorp.com immediately
3. Do not delete any evidence
4. Document what happened

## Compliance
- Complete annual security training (required within first 30 days)
- Violations may result in disciplinary action up to termination
- Security audits conducted quarterly

**Questions?** Contact it-security@techcorp.com or visit #security-help on Slack.
